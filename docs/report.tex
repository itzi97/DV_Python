\section{Dataset}
% Brief dataset desc + questions overview

\par
Data for this project was required to not have been selected by a different 
student. To minimize the risk of this happening, activity data from a 
\href{https://www.garmin.com/en-US/p/628939/}{Garmin Forerunner 245} was chosen, 
reducing the chances of another student having the same dataset.

The data set, located in \lstinline{./data/activities.csv}, consists of:

\begin{itemize}
  \item 31 columns \& 180 rows.
  \item Data spanning from: 21 February 2021 - 30 March 2026 (5 years).
  \item Varying data types (integers, floats, dates, time spans, locations, strings).
\end{itemize}

\par
Having 31 columns allows for many different questions to be asked about the 
dataset, allowing it to be explored in multiple ways if desired. The limited
number of rows (180) whist still being a decent amount, allows it's content to
be analyzed in a more hands on way, being able to be more critical about each
specific row and actively engage in processing in a more granular fashion. The
dates spanning 5 years back allows for a story to be told through time, and
analyze factors like consistency. Lastly, the varying data types allows for
different types of visualizations that require different data types to be used
such as maps (with location).

\subsection{Analysis}

\par
Before asking questions about the dataset, the dataset was further analyzed
using both the \lstinline{df.info()} function to check how the data was 
imported into the pandas dataframe.

\par
A small AI analysis was invoked using qwen2.5:14b for the prompt which was the
following:

\begin{quote}
  Read the linked CSV file and analyze it, create a table with the following 
  fields:
  \begin{itemize}
    \item N\cdeg: Column number.
    \item Column: Column name.
    \item Description: Content description (what the data in the column 
      usually looks like), excluding null indicators).
    \item Null indicator: Indicator that seems to be used for when the data is
      null.
    \item Data type: Suggested data type to be used when reading CSV in Pandas.
  \end{itemize}
\end{quote}

\par
The table generated was then used as a starting point although it's content was
scrutinized and interpreted correctly, assessing whether or not the AI had 
hallucinated when generating it. Some important conclusions reached with the 
help of this analysis were:

\begin{itemize}
  \item Some columns weren't really needed or used (such as \textit{Favorite}).
  \item \lstinline{--} was used as a null indicator on most columns.
  \item Correct data types for each column.
\end{itemize}

\subsection{Questions}

\par
Having analyzed and processed the data, many rows and columns were cleaned up
from the dataset and columns were changed to the correct format. Having the
data in this format, allowed for questions to be asked since now there is a
better idea of the data at hand. The following questions were therefore asked
taking into account the current state of the data:

\begin{enumerate}
  \item What kind of activities are being recorded?\label{q1}
  \item Are the recorded activities showing enough weekly activity?\label{q2}
  \item Where are activities being recorded, and how do activity patterns differ 
    by location?\label{q3}
\end{enumerate}

\par
Each question and why they were chosen i explained below:

\subsubsection{Question~\ref{q1}: Activity Composition}
\par
Question~\ref{q1} is a good starting question because it dives into 
categorizing data into different types of activity which is a good starting
point to understand what type of data is being recorded and later explored in
other questions.

\subsubsection{Question~\ref{q2}: Weekly Activity Assessment}
\par
Question~\ref{q2} proceeds naturally from question~\ref{q1} as it benchmarks
personal fitness against WHO guidelines\parencite{WHO2020} (150 min moderate 
or 75 min vigorous per week). This question requires aggregating Time by week 
and classifying intensity via Avg HR threshholds, enabling trend analysis over 5 
years.

\subsubsection{Question~\ref{q3}: Geographic Patterns}
\par
Question~\ref{q3} adds spatial context by extracting locations from Title,
mapping activity density and compraing metrics by place. It also incentivizes
data to be used in a more creative way, not just looking at the given fields, 
but extracting a whole different field type to create a new field type such as
location in this case from just a title.
